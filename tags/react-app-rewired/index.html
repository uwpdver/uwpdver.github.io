Tailwindcss

tailwindcss 是一个最近很流行的 css 框架，它提供了很多常用的工具类名，使用起来可以让前端开发体验更加方便，快捷。

PostCSS

tailwindcss 使用作为 postcss 的 plugin 的方式安装，那什么是 postcss? 在 postcss 官方文档中给出的解释为：一个使用 JavaScript 转换 CSS 的工具，它相当于提供一个扩展 CSS 的方式，借助它可以对 CSS 做一些超过原生 CSS 能力的更复杂的事情，常见的用例包括在 CSS 类名后面加上全局唯一的后缀以避免样式污染，或者自动为 CSS 属性增加浏览器前缀，对 CSS 的语法检测等等。这些拓展能力，可以通过 postcss 的插件的方式来配置，可以根据需要增加自己需要的插件，而 tailwindcss 就是一个 postcss 的插件。

如何在 create-react-app 生成的项目中安装

那如何安装 tailwindcss 的问题就变成了如何安装 postcss 的插件的问题。postcss 并不是一个内置的功能，他同样也是一个三方库，可以通过 npm 安装，安装之后，可以通过配置文件对 postcss 进行配置，相关的配置方法很简单，在网上很容易找到答案，但是对于通过 create-react-app 的方式创建的项目，create-react-app 不允许覆写 postcss 的配置，所以需要借助别的库。官方文档中给出的方法是通过 craco 这个库来改写 webpack 的 postcss 相关的配置。但是对于有些已经存在的项目，可能已经使用了另一种方式去改写 webpack 的配置，通过 react-app-rewired + customize-cra 配置，官方并没有给出相关的教程。但是通过观察 craco 的方式的配置可以看出来，实际上就是为 postcss.plugins 这一配置项中添加 tailwindcss 和 autoprefixer 这两个模块。
